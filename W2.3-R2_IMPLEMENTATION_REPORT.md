## Executive Summary

W2.3-R2 wires post-turn context accumulation into the canonical turn execution flow. Completed turns now automatically:
1. Derive `ShortTermTurnContext` from the execution result
2. Initialize `SessionHistory` (bounded to max 100 entries) if not present
3. Create a `HistoryEntry` from short-term context
4. Add the entry to session history (with FIFO trimming when full)

All context is created—even for failed or structurally invalid turns—ensuring complete auditability of the session's turn sequence.

#### `_accumulate_turn_context()` Helper (lines ~260-301)

```python
def _accumulate_turn_context(
    session: SessionState,
    result: TurnExecutionResult,
    prior_scene_id: str | None = None,
) -> None:
    """Create short-term context and add to session history.

    Local imports prevent circular dependency with short_term_context module.
    """
    # Local imports to avoid circular dependency
    from app.runtime.session_history import HistoryEntry, SessionHistory
    from app.runtime.short_term_context import build_short_term_context

    # Initialize session history if not present (bounded to 100 entries)
    if session.context_layers.session_history is None:
        session.context_layers.session_history = SessionHistory(max_size=100)

    # Derive short-term context from execution result
    short_term_context = build_short_term_context(result, prior_scene_id=prior_scene_id)
    session.context_layers.short_term_context = short_term_context

    # Add entry to bounded history (FIFO trimming when full)
    if isinstance(session.context_layers.session_history, SessionHistory):
        history_entry = HistoryEntry.from_short_term_context(short_term_context)
        session.context_layers.session_history.add_entry(history_entry)
```

## Runtime Turn Execution Flow (Post-R2)

```
execute_turn(session, turn_number, decision, module):
    ├─ prior_scene_id = session.current_scene_id  [← NEW: Capture for transition detection]
    │
    ├─ VALIDATION PHASE
    │  └─ validate decision structure, references, triggers
    │
    ├─ CONSTRUCTION PHASE
    │  └─ build proposed state deltas
    │
    ├─ APPLICATION PHASE
    │  └─ apply deltas to canonical_state (with guard filtering)
    │
    ├─ FINALIZATION PHASE
    │  └─ update session fields (current_scene_id, turn_counter, etc)
    │
    ├─ ✅ SUCCESS or ❌ FAILURE
    │
    ├─ ALL PATHS (success and failure)
    │  └─ _accumulate_turn_context(session, result, prior_scene_id)  [← NEW]
    │     ├─ build_short_term_context(result, prior_scene_id)
    │     │  └─ ShortTermTurnContext: turn_number, scene_id, triggers, deltas, guard_outcome, scene_changed, conflict_pressure
    │     │
    │     ├─ Initialize session.context_layers.session_history if None
    │     │  └─ SessionHistory(max_size=100)
    │     │
    │     ├─ Update session.context_layers.short_term_context = ShortTermTurnContext
    │     │
    │     └─ HistoryEntry.from_short_term_context()
    │        └─ session_history.add_entry(entry) [← FIFO trimming when size > max_size]
    │
    └─ return result
```

## Accumulation Behavior: All Outcomes

### Successful Turn (guard_outcome="accepted")
- ✅ Short-term context created with all accepted_delta_targets
- ✅ History entry created and added to SessionHistory
- ✅ short_term_context updated to reflect latest turn

### Rejected Turn (guard_outcome="rejected")
- ✅ Short-term context created with rejected_delta_targets
- ✅ No accepted deltas applied to canonical_state
- ✅ History entry still created and accumulated
- ✅ Session remains valid for next turn

### Partially Accepted Turn (guard_outcome="partially_accepted")
- ✅ Short-term context created with both accepted and rejected delta lists
- ✅ Only accepted deltas applied to canonical_state
- ✅ History entry still created and accumulated
- ✅ Context reflects which deltas passed/failed validation

### Structurally Invalid Turn (guard_outcome="structurally_invalid")
- ✅ Short-term context created with empty trigger/delta lists
- ✅ No deltas attempted on canonical_state
- ✅ History entry still created
- ✅ Audit trail shows attempted invalid decision

### System Error (e.g., exception during execution)
- ✅ Short-term context created for the failed turn
- ✅ History entry still created
- ✅ Session remains recoverable for next turn
- ✅ Error cause visible in context/history

**Key Principle**: SessionHistory is the canonical audit trail. All turns—successful, failed, or erroneous—are accumulated, ensuring complete transparency of decision sequence.

## SessionHistory Bounded Storage

**Capacity**: max_size=100 entries (configurable per SessionHistory instance)

**Retention Policy**: FIFO (First-In-First-Out)
- When full and new entry added: oldest entry discarded
- Newest entries always preserved
- Circular buffer pattern prevents unbounded memory growth

**Example Scenario**:
```python
# Day 1-5 session: 500 turns executed
session.context_layers.session_history.max_size == 100
session.context_layers.session_history.size == 100
session.context_layers.session_history.entries[0].turn_number == 401  # Oldest retained
session.context_layers.session_history.entries[-1].turn_number == 500  # Latest
```

## Circular Import Prevention

**Challenge**: `turn_executor` needs to import `SessionHistory` and `build_short_term_context`, but `short_term_context` imports `TurnExecutionResult` from `turn_executor`, creating circular dependency.

**Solution**: Use local imports inside `_accumulate_turn_context()` function
```python
def _accumulate_turn_context(session, result, prior_scene_id):
    # Import at function scope, not module scope
    from app.runtime.session_history import SessionHistory, HistoryEntry
    from app.runtime.short_term_context import build_short_term_context

    # Now safe to use imported types
    ...
```

**Why This Works**:
- Module-level imports execute at file load time (creates cycle)
- Function-level imports execute only when function is called
- By the time `_accumulate_turn_context()` is called, both modules are fully initialized
- No circular dependency at import time
