## Integration Option Chosen: Option B (Wrapper Struct)

### Rationale
Option B was chosen over Option A (direct fields) because:

1. **Modularity**: W2.3 layers stay grouped together, not scattered across SessionState
2. **Maintainability**: SessionContextLayers can be extended without growing SessionState field list
3. **Clarity**: Clear boundary between core session fields and context layers
4. **Future-proofing**: Easy to add W2.4, W2.5 context layer wrappers later
5. **Readability**: SessionState remains concise (single `context_layers` field vs. 5 separate fields)

### Design Pattern
```
SessionState
├── session_id, module_id, current_scene_id, ... [core fields]
└── context_layers: SessionContextLayers
    ├── short_term_context (W2.3.1)
    ├── session_history (W2.3.2)
    ├── progression_summary (W2.3.3)
    ├── relationship_axis_context (W2.3.4)
    └── lore_direction_context (W2.3.5)
```

## SessionContextLayers Model

```python
class SessionContextLayers(BaseModel):
    """W2.3 memory and context layers for a session.

    Contains the five bounded context layers for longer-session coherence.
    All fields are optional (None) until explicitly derived/accumulated.
    """

    short_term_context: Optional[Any] = None       # W2.3.1
    session_history: Optional[Any] = None          # W2.3.2
    progression_summary: Optional[Any] = None      # W2.3.3
    relationship_axis_context: Optional[Any] = None # W2.3.4
    lore_direction_context: Optional[Any] = None   # W2.3.5
```

**Key Design Decisions**:
- Type is `Optional[Any]` to allow any W2.3 model (ShortTermTurnContext, SessionHistory, etc.)
- All fields default to `None` until derived or accumulated
- Serializable to JSON for persistence (future work)
- Clean API: `session.context_layers.session_history`

## Canonical Runtime State Shape

### SessionState Structure (Post-R1)

```python
session = SessionState(
    # Core session identity
    session_id="abc123",
    module_id="god_of_carnage",
    module_version="1.0.0",
    current_scene_id="phase_1",

    # Session status
    status=SessionStatus.ACTIVE,
    turn_counter=5,

    # World state
    canonical_state={"characters": {...}, ...},

    # Execution config
    execution_mode="ai",
    adapter_name="claude_story",
    seed=None,

    # Timestamps
    created_at=datetime(...),
    updated_at=datetime(...),

    # Extensibility
    metadata={},

    # W2.3 Memory and context layers
    context_layers=SessionContextLayers(
        short_term_context=None,  # Will hold ShortTermTurnContext when derived
        session_history=None,      # Will hold SessionHistory when accumulated
        progression_summary=None,  # Will hold ProgressionSummary when derived
        relationship_axis_context=None,  # Will hold RelationshipAxisContext when derived
        lore_direction_context=None,     # Will hold LoreDirectionContext when derived
    )
)
```

### Access Patterns

**Read/write context layers**:
```python
# Access
session.context_layers.session_history

# Update
session.context_layers.session_history = history_instance

# Reset
session.context_layers = SessionContextLayers()
```
