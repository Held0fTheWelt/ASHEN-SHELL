# Despaghettify Workstream Policy

Despaghettify is adopted as the structural-governance suite for implementation continuation.

## Role

Despaghettify is not a runtime engine.
It is the package discipline used to reduce structural chaos while runtime work proceeds.

## Required uses

For any medium or large implementation change:

1. open or identify a bounded workstream,
2. capture pre-state evidence,
3. perform the implementation in slices,
4. capture post-state evidence,
5. compare residual hotspots,
6. keep unresolved structural risks visible.

## Why this matters in World of Shadows

This project has many interacting surfaces:

- runtime truth,
- AI orchestration,
- memory pipelines,
- service boundaries,
- authoring and admin governance,
- player-facing delivery.

Without explicit workstream discipline, changes can easily become implicit, cross-cutting, and hard to validate.

Despaghettify is therefore adopted here as an anti-regression and anti-drift structure layer.
