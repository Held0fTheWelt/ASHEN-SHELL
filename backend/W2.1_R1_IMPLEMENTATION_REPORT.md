## Canonical Runtime Entry Point

**`dispatch_turn()` in `backend/app/runtime/turn_dispatcher.py`**

This is now the single authoritative entry point for turn execution. All turn execution requests should go through this function.

**Signature:**
```python
async def dispatch_turn(
    session: SessionState,
    current_turn: int,
    module: ContentModule,
    *,
    mock_decision_provider: callable | None = None,
    ai_adapter: StoryAIAdapter | None = None,
    operator_input: str = "",
) -> TurnExecutionResult
```

**Selection logic:**
1. Check `session.execution_mode`
2. If "ai": require `ai_adapter`, call `execute_turn_with_ai()`
3. If "mock" or default: call `execute_turn()` with optional decision provider
4. Return `TurnExecutionResult` (identical structure from both paths)

---

## Path Selection Mechanism

### Selection Rule

**Session-level configuration:**

```python
session.execution_mode = "mock"  # or "ai"
```

The dispatcher checks this field and routes accordingly.

### How Selection Works

```python
execution_mode = session.execution_mode.lower() if session.execution_mode else "mock"

if execution_mode == "ai":
    # AI path
    return await execute_turn_with_ai(session, current_turn, ai_adapter, module, ...)
else:
    # Mock path (default)
    return await execute_turn(session, current_turn, decision, module)
```

### Why This Approach

- **Explicit:** Selection is visible and testable
- **Simple:** No complex heuristics or hidden logic
- **Flexible:** Can be extended to other execution modes later
- **Backward compatible:** Default is "mock" (existing behavior)
