# 03 — Five-Layer Execution Architecture

## Layer 1 — Data and Memory
Canonical state, domain-aware memory records, partitions, temporal validity, provenance, lineage, and retrieval indexes.

## Layer 2 — Semantic
Interpret player input, infer surface and deep intent, retrieve task-specific memory context, generate candidate continuations, validate legality and partition safety.

## Layer 3 — Narrative
Arc state, beat selection, mystery rationing, callbacks, foreshadowing, pacing, scene energy, thematic and tonal coherence.

## Layer 4 — Emotional
Player comprehension, frustration, overload, boredom, agency, support pressure, pacing moderation, and emotional validation.

## Layer 5 — Consciousness
Story-health vector, meta-strategy, degraded-mode supervision, callback debt, mystery integrity, escalation versus support.

## Execution order

### Top-down
Consciousness → Emotional → Narrative → Semantic

### Bottom-up
Semantic validation → Narrative validation → Emotional validation → Consciousness validation → Commit

Emotional and consciousness layers are explicit runtime participants with their own state and outputs.
