# 04 — Player Experience and Accessibility Specification

## Experience goals

The player experience must feel responsive, readable, choice-preserving, emotionally legible, supportive without over-explaining, and clearly separated from operator internals.

## Required controls

- free-text input
- send
- recap
- help / hint
- settings
- save session
- load session
- transcript navigation
- accessibility menu
- optional undo / redo only where canon law permits it

## Interaction states

- idle
- typing
- sending
- processing
- degraded processing
- result received
- error
- safe-blocked with explanation
- operator-assisted if live intervention occurs

## Accessibility baseline

- keyboard-only navigation
- semantic headings and landmarks
- transcript entries readable by screen readers
- reduced motion option
- high contrast mode
- font scaling up to 200%
- dyslexia-friendly mode
- color-independent status indicators

See `ui_mockups/` for concrete mockups.
