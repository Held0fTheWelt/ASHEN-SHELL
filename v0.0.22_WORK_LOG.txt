FORUM TEST SUITE (Backend/tests/test_forum_api.py - comprehensive)
  ✓ STRUCTURE SOUND: 27 tests covering all major workflows
  ⚠ EXECUTION BLOCKED: SQLAlchemy DetachedInstanceError affects 17 tests
    Root Cause: Test fixtures return objects after app context exits. When route
    handler tries to access thread.id, SQLAlchemy tries to refresh from session
    and fails because session is detached.
    Solution Pattern Needed: Either merge objects back or use fresh context lookup
    Example Fix:
      Before: thread = create_thread(...); return thread.id  # detached!
      After: thread = create_thread(...); db.session.merge(thread); return thread.id
             OR: thread_id = create_thread(...).id; inside context; return thread_id

OVERALL COVERAGE
  Current: 33% (blocked by forum test fixture issue)
  Gate: 85% (pytest.ini fail-under=85)
  Status: Forum-specific tests are well-written; fixture scope is the blocker
