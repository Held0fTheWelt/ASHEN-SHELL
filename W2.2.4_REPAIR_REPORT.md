## Option Chosen: Option A

**Option A: Trigger-Aware Validation-Time Checks**

**Rationale:**
1. `MockDecision` already contains `detected_triggers` field (available at validation time)
2. Validation-time and execution-time can use the same trigger evidence
3. Eliminates the silent asymmetry; behavior becomes coherent
4. Minimal change: extract and pass triggers from decision object

**Implementation:**
```python
# At validation time:
decision_triggers = getattr(decision, "detected_triggers", [])
legality_decision = SceneTransitionLegality.check_transition_legal(
    current_scene_id, decision.proposed_scene_id, module,
    session=session, detected_triggers=decision_triggers  # Use actual triggers
)

# At execution time (already does this):
legality_decision = SceneTransitionLegality.check_transition_legal(
    session.current_scene_id,
    mock_decision.proposed_scene_id,
    module,
    session=session,
    detected_triggers=mock_decision.detected_triggers  # Use actual triggers
)
```

Both now use the same trigger evidence, making behavior coherent.

## Validation-Time Contract (Now Explicit)

**Old Contract (Implicit/Ambiguous):**
- Scene transitions validated with `detected_triggers=None`
- Conditional transitions might appear legal at validation time but fail at execution
- Asymmetry was silent and undocumented

**New Contract (Explicit and Coherent):**
- Scene transitions validated with actual `decision.detected_triggers`
- Validation and execution behavior matches
- If validation says "legal", execution will also say "legal" (given same triggers)
- No silent asymmetries

**Guarantees:**
1. Unconditional transitions: Always legal at both validation and execution time
2. Conditional transitions: Legal at both times iff required triggers are detected
3. No transitions are "silently" different between validation and execution

- **Conditional with triggers:** Legally triggered at both times
- **Conditional without triggers:** Not triggered at both validation and execution (now coherent)
