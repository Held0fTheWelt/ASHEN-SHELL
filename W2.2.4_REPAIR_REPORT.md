## Problem Statement

The initial W2.2.4 implementation (commit `a4d2808`) introduced coherent legality rules via `SceneTransitionLegality`, but created a hidden drift issue:

### The Drift Scenario

**Validator path (at decision validation time):**
```python
legality_decision = SceneTransitionLegality.check_transition_legal(
    current_scene_id, decision.proposed_scene_id, module,
    session=session, detected_triggers=None  # No trigger evidence available
)
```

**Derivation path (at next situation time):**
```python
legality_decision = SceneTransitionLegality.check_transition_legal(
    current_scene_id, transition.to_phase, module,
    session=session, detected_triggers=["detected_triggers_here"]  # Has evidence
)
```

### What Went Wrong

For a module with **only a conditional transition** from scene A to scene B (requiring trigger "unlock"):

1. **Validator:** Checked with `detected_triggers=None`
   - `_check_transition_conditions()` returned `False, "Conditions undefined but detected_triggers not provided"`
   - This was **correct** (cannot evaluate without evidence)

2. **But:** The error message from `check_transition_legal()` returned:
   - `"All transitions from A to B require unmet conditions"`
   - This was **misleading** — not "unmet", but "unevaluated"

3. **Result:** Error message obscured the actual legality contract:
   - Validator couldn't prove conditions were met (no evidence)
   - Derivation might later prove conditions **were** met (with actual triggers)
   - But both paths should reach the same decision for the same scenario

### The Incoherence

While both paths ultimately rejected the transition (validator: no evidence; derivation: triggers not detected), the error messages created confusion about the **legality contract**:

- Validator error: "require unmet conditions" (when triggers not available)
- Derivation error: "Missing conditions: [list]" (when triggers detected but don't include required one)
- These are conceptually different, but the error messages made them look the same

---

## Solution

### Root Cause Fix

Modified `check_transition_legal()` to **preserve the specific rejection reason** from `_check_transition_conditions()` instead of collapsing all rejections to a generic message.

**Before:**
```python
# All transitions require conditions that aren't met
return SceneLegalityDecision(
    allowed=False,
    reason=f"All transitions from '{from_scene}' to '{to_scene}' require unmet conditions"
)
```

**After:**
```python
# All transitions require conditions that aren't met
# Use specific rejection reason if available (e.g., "Conditions undefined but detected_triggers not provided")
if last_rejection_reason:
    return SceneLegalityDecision(
        allowed=False,
        reason=f"Transition from '{from_scene}' to '{to_scene}' cannot proceed: {last_rejection_reason}"
    )
else:
    return SceneLegalityDecision(
        allowed=False,
        reason=f"All transitions from '{from_scene}' to '{to_scene}' require unmet conditions"
    )
```

### What This Achieves

Now the error message explicitly states **why** the transition is rejected:

1. **Validator with `detected_triggers=None`:**
   - Reason: `"Transition from A to B cannot proceed: Conditions undefined but detected_triggers not provided"`
   - Clearly indicates: cannot evaluate without trigger evidence

2. **Derivation with `detected_triggers=[]`:**
   - Reason: `"Transition from A to B cannot proceed: Missing conditions: ['unlock']"`
   - Clearly indicates: conditions present but not satisfied

3. **Derivation with `detected_triggers=['unlock']`:**
   - Reason: `"Transition from A to B is legal"`
   - Transition allowed

**All three paths are now semantically coherent.** Each message explains the actual state.

## Canonical Legality Rule

**Conditional Scene Transition Legality (Final):**

A conditional transition from scene A to scene B is legal **if and only if**:

1. **Scene B exists** in the module
2. **A transition** from A to B is defined (matching transition_id)
3. **Either:**
   - The transition has **no trigger conditions** (unconditional), OR
   - The transition requires conditions **AND** we have trigger evidence **AND** all required conditions are detected

**Rejection reasons are explicit:**
- Missing trigger evidence: "Conditions undefined but detected_triggers not provided"
- Unmet trigger conditions: "Missing conditions: [list of missing]"
- Transition doesn't exist: "Scene X not reachable from Y"
- Target scene invalid: "Scene X not in module"
