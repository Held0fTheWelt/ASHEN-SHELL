**Risk 1: Trigger Applicability Is a Placeholder**
- **What looks complete:** Code exists at reference_policy.py:220-224
- **What's not complete:** Returns True for all triggers; doesn't actually check applicability
- **Why it doesn't matter:** Existence validation catches invalid triggers; applicability refinement is deferred scope
- **How to avoid:** Be aware that trigger_applicable_in_scene() is not implemented and will need W2.2.5 work

**Risk 2: validate_action_structure() Optional Parameters**
- **What looks complete:** signature accepts module and session
- **What could break:** Old callers without module/session still work (backward-compatible) but won't validate references
- **Why it's OK:** The real runtime path provides module/session, so it validates; legacy code without them just skips reference validation (which was the pre-integration behavior)

**Risk 3: Scene Reachability Model Assumptions**
- **What's assumed:** phase_transitions contains valid transition definitions
- **What could break:** If phase_transitions is malformed or missing
- **Mitigation:** Code safely returns False if phase_transitions doesn't exist (line 195-196)
