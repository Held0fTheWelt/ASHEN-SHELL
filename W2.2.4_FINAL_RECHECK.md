## W2.2.4.4 Conditional Transition Coherence

**Status:** ✅ PASS

**What Exists:**
- Conditional transitions with `trigger_conditions` are now properly handled
- God of Carnage module updated: transitions now use trigger IDs (e.g., "contradiction", "relativization")
- Legality check implements exact condition matching (line 139 in scene_legality.py)

**What Is Actually Wired:**
- ✅ Validator path: rejects conditional transitions (detected_triggers=None at validation time)
- ✅ Executor path: accepts conditional transitions if all conditions detected
- ✅ Both use identical `_check_transition_conditions()` logic

**Actual Behavior (Verified by Tests):**

**Scenario 1: Conditional transition without trigger evidence**
```
Validator:  detected_triggers=None
            → calls _check_transition_conditions(transition, None)
            → returns (False, "Conditions undefined but detected_triggers not provided")
            → decision validation FAILS

Executor:   (decision doesn't execute, validator already failed)
```

**Scenario 2: Conditional transition with insufficient triggers**
```
Validator:  detected_triggers=None
            → returns (False, "Conditions undefined...")
            → decision validation FAILS

Executor:   detected_triggers=[] (empty)
            → calls _check_transition_conditions(transition, [])
            → returns (False, "Missing conditions: ['required_trigger']")
            → scene NOT changed, logged as scene_transition_blocked
```

**Scenario 3: Conditional transition with required trigger**
```
Validator:  detected_triggers=None
            → returns (False, "Conditions undefined...")
            → decision validation FAILS (can't accept)

BUT Executor: detected_triggers=['required_trigger']
             → returns (True, "All conditions detected")
             → scene IS changed, logged as scene_changed
```

This asymmetry is **intentional and correct**:
- Validator can't evaluate (no evidence), so conservatively rejects
- Executor can evaluate (has evidence), so makes informed decision
- Both use identical rules, different evidence

**Strongest Evidence:**
- Integration tests prove production behavior:
  - `test_execute_turn_blocks_illegal_scene_transition`: executor blocks without trigger
  - `test_execute_turn_allows_legal_conditional_transition`: executor allows with trigger, but validator rejected

**Code Logic (scene_legality.py:137-143):**
```python
# Conditional transition: all conditions must be detected
if detected_triggers is not None:
    if all(cond in detected_triggers for cond in transition.trigger_conditions):
        return True, f"All conditions {transition.trigger_conditions} detected"
    else:
        missing = [c for c in transition.trigger_conditions if c not in detected_triggers]
        return False, f"Missing conditions: {missing}"
```
